# Portfolio360 – نسخه اندروید (اسکلت اولیه)

این نسخه‌ی جاوا/اندرویدِ ابزار Portfolio360 است. در این مرحله فقط **اسکلت** پروژه ساخته شده:
۹ تب متناظر با تب‌های نسخه Streamlit وجود دارد، اما محتوای هرکدام فعلاً یک صفحه placeholder است.

## چطور اجرا/تست کنم؟ (بدون ترمینال، فقط با موبایل)

1. این پوشه را در یک ریپوی گیت‌هاب جدید push کنید (یا محتوایش را داخل ریپوی `coverd-call` در یک شاخه/پوشه جدا قرار دهید).
2. به محض push شدن روی شاخه `main`، ورک‌فلو `.github/workflows/android-build.yml` خودکار اجرا می‌شود و پروژه را build می‌کند.
3. به تب **Actions** ریپو بروید → آخرین اجرا را باز کنید → پایین صفحه، بخش **Artifacts** یک فایل به اسم
   `portfolio360-debug-apk` پیدا می‌کنید.
4. آن را از گوشی دانلود کنید (فایل zip است، داخلش `app-debug.apk`)، و روی گوشی نصب (sideload) کنید.
   (نیاز به فعال کردن «نصب از منابع ناشناس» دارید.)

## ساختار فعلی

- `MainActivity.java` – تب‌ها را با `TabLayout` + `ViewPager2` می‌سازد.
- `TabPagerAdapter.java` – برای هر تب یک فرگمنت می‌سازد.
- `PlaceholderFragment.java` – فرگمنت موقت که در آینده هرکدام با منطق واقعی جایگزین می‌شود.

## مراحل بعدی (جلسات آینده)

هر جلسه، منطق ۱ تا ۲ تب به‌صورت کامل با جاوا پیاده‌سازی می‌شود:
- محاسبات Black-Scholes/Greeks (کلاس‌های محاسباتی خالص جاوا، بدون کتابخانه خارجی)
- اتصال به API قیمت (Derive.xyz / Deribit DVOL) با OkHttp
- نمودارها با MPAndroidChart (معادل Plotly)
- منطق پیچیده‌تر (Monte Carlo، Black-Litterman) به‌صورت مرحله‌ای

## نکته مهم

این پروژه هرگز روی سیستم واقعی Android Studio build نشده (چون محیط توسعه من ترمینال/شبکه اندرویدی ندارد).
احتمال دارد اولین build با یک خطای کوچک (نسخه پلاگین، وابستگی) مواجه شود.
اگر لاگ خطای Actions را برایم بفرستید، فوراً رفعش می‌کنم.
